# Laboratorio DEBUG en DOSBox

## Descripción
En este laboratorio se utilizó DOSBox para ejecutar el programa DEBUG y analizar registros, memoria y código ensamblador.

## Checkpoint 1
Se utilizó el comando R para visualizar los registros del procesador. Se observó que AX, BX, CX y DX estaban en cero.

## Checkpoint 2
Se utilizó el comando F para llenar memoria con el patrón AB CD EF y el comando D para visualizarlo. Se observó la repetición del patrón.

## Checkpoint 3
Se utilizó el comando A para crear instrucciones en ensamblador y el comando U para verificarlas.

## Conclusión
El laboratorio permitió comprender el funcionamiento interno del procesador, la memoria y el lenguaje ensamblador.
